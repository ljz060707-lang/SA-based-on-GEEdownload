## 2. Agent-First Model Benchmark 设计与实施

### 概述

设计并实现了训练后模型权重对比的标准化 benchmark 系统。经 Claude + Codex 三轮交叉审阅，确定 agent-first 设计方向：主输出为 `summary.json`（机器可读结论），agent 跑完直接告诉用户哪个权重更好。

### 设计过程

1. **Claude 初版**: 轻量 YAML 配置 + 6 个 benchmark grids + Markdown 报告
2. **Codex 第一版**: Suite 分层 (smoke/primary/diagnostic/secondary)、run manifest、三层聚合、但 CLI 笨重（每次手写路径）
3. **Codex agent-first 提案**: summary.json 主输出、自动 verdict、TL;DR 摘要
4. **交叉审阅融合**:
   - 目录层级折中 → `benchmark_<run_id>_<tag>/`（扁平但含 run_id）
   - Checksum 默认 auto（缓存机制）
   - 零参数启动 + 显式 banner

### 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 主输出 | summary.json | agent 读 JSON 比解析 Markdown 可靠 |
| 自动 verdict | improved/regressed/flat/mixed/failed | agent 不必自己解释数字 |
| 排名依据 | primary suite (26 grids) F1 micro | 与 exp003 口径一致 |
| Suite 分层 | smoke/primary/diagnostic/secondary | 区分用途，诊断集不参与排名 |
